üí° IMPORTANT NOTES
================================================================================

Security:
- NEVER commit .env files to git
- Use .env.example as template
- Set all secrets in dashboard
- Rotate secrets periodically

Deployment Order:
- Deploy BACKEND first (frontend depends on it)
- Get backend URL from Render
- Then deploy FRONTEND with backend URL
- Update CORS after getting frontend URL

Database:
- PostgreSQL is created on Render
- Migrations run automatically
- Backups enabled by default
- Connection pooling recommended

Connection:
- Frontend URL must be in backend CORS_ORIGINS
- Backend URL must be in frontend NEXT_PUBLIC_API_URL
- Test: curl https://your-backend.onrender.com/health
